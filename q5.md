# Q5 — Standardizing numeric vectors, and why it matters before decomposing a data matrix

## What standardization is

Standardization (also called z-score normalization) rescales each feature — each column of the data matrix — so that it has mean 0 and standard deviation 1. For a feature $j$ with values $x_{1j}, x_{2j}, \dots, x_{dj}$ across all $d$ samples, with mean $\mu_j$ and standard deviation $\sigma_j$:

$$
z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j}, \qquad
\mu_j = \frac{1}{d}\sum_{i=1}^{d} x_{ij}, \qquad
\sigma_j = \sqrt{\frac{1}{d}\sum_{i=1}^{d}(x_{ij} - \mu_j)^2}
$$

After this transformation, every column lives on the same footing: values are unitless, centered at zero, and measured in "number of standard deviations away from the mean" rather than in whatever raw units the feature originally had.

## Why this matters before decomposing the matrix

SVD and PCA both work by finding the directions in the data that capture the most variance. That's the entire mechanism — the first singular vector points along whatever direction the data spreads out the most, the second one is orthogonal to it and points along the next most-spread-out direction, and so on.

The problem is that "variance" is scale-dependent. If one column in your data matrix is measured in a unit that happens to produce large numbers (say, values ranging from 0 to 10,000) and another column is measured in a unit that produces small numbers (say, values ranging from 0 to 1), the first column will have enormously larger variance purely as an artifact of its units — not because it's actually more informative. When you feed this into SVD, the top singular vectors will end up pointing almost entirely along that first, large-scale column, simply because that's where the raw numerical variance happens to sit. The decomposition ends up describing which feature had the biggest units, not which features are actually correlated with each other or which patterns are genuinely dominant in the data.

Standardizing removes this artifact. By forcing every feature onto the same scale (mean 0, standard deviation 1) before decomposing, you make sure that SVD is responding to how the *features relate to each other* — their genuine covariance structure — rather than to arbitrary differences in units or magnitude. Without this step, a decomposition can be technically correct while being practically misleading, since it's letting one or two large-scale features dominate the entire result.

## When it's less critical

If all the features in your matrix are already naturally on a comparable scale — for example, a matrix of TF-IDF weights, or word counts, where every entry is measured the same way and no single column has an inherently different unit or range — the case for standardization is weaker, because the scale-dominance problem described above doesn't really arise. But whenever a data matrix combines heterogeneous measurements (different units, wildly different ranges), standardizing before SVD/PCA isn't optional — it's a prerequisite for the decomposition to reflect real structure in the data rather than an accident of measurement units.
