# Q2 — The Elbow Point approach for choosing the rank in Truncated SVD

When you compute the SVD of a matrix $A_{d\times n} = U\Sigma V^T$, the diagonal of $\Sigma$ holds the singular values $\sigma_1 \ge \sigma_2 \ge \dots \ge \sigma_r \ge 0$, sorted from largest to smallest. Each singular value tells you how much of the matrix's "energy" — how much of its structure — is captured by the corresponding pair of singular vectors. The first singular value corresponds to the strongest, most dominant pattern in the data; the last ones correspond to the faintest, most marginal patterns.

Truncated SVD keeps only the first $k$ of these singular values (and their matching vectors) and throws the rest away, giving a rank-$k$ approximation $A_k = git@github.com:mohammadrezah7/Linear-Algebra-project.gitU_k \Sigma_k V_k^T$. The whole question then becomes: what value of $k$ should we pick?

## The idea behind the elbow point

If you plot the singular values on the y-axis against their rank index $i = 1, 2, 3, \dots$ on the x-axis, the curve almost always has a distinctive shape for real-world data: it drops very steeply for the first few values, and then the drop slows down and the curve flattens into a long, gently-decaying tail.
v
That bend — the spot where the curve stops falling steeply and starts leveling off — is the "elbow" (also called the "knee"). Visually it looks like the inside of a bent arm.

The reasoning for cutting there is this: the singular values before the elbow are large and clearly separated from each other, which means they represent genuinely significant, structural patterns in the data — the "signal." The singular values after the elbow are small, close to each other, and decaying slowly, which is the signature of noise or negligible fine-grained detail rather than meaningful structure. So choosing $k$ to be the index right at the elbow is a way of saying: "keep everything that looks like real structure, and discard everything that looks like noise."

## Why this is a good tradeoff

Cutting at the elbow gives you a rank $k$ that is usually much smaller than the true rank $r$ of the original matrix, but still captures the overwhelming majority of the matrix's information — because the discarded singular values are small, and (as we'll see in Q3) the reconstruction error is directly tied to the size of the singular values you throw away. In other words, the elbow point is roughly the point of diminishing returns: past it, you're spending more dimensions to capture less and less extra information, so it's not worth keeping them around.

In practice this is done visually — plot the singular values, look for where the curve "kinks," and pick that index as $k$. It's a heuristic rather than an exact rule (the bend isn't always sharp), but it's fast, intuitive, and works well when the singular values decay quickly, which is the typical case for text data like the term-document matrices used in LSI.
