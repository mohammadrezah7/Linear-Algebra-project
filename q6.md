# Q6 — Randomized SVD for large matrices

## The problem it solves

Computing a full, exact SVD of a matrix $A$ of size $m \times n$ costs roughly $O(mn \cdot \min(m,n))$ time. For a small term-document matrix that's fine, but the moment you're working with a matrix with hundreds of thousands of rows and columns — which is exactly the regime LSI lives in once you scale past a toy dataset — an exact SVD becomes painfully slow or simply infeasible to compute.

The saving grace is that in applications like LSI you almost never need the *full* SVD anyway. You only want the top $k$ singular values and vectors, because that's all Truncated SVD is going to keep. Randomized SVD is built exactly for this situation: it skips computing the full decomposition and instead directly targets a good approximation of just the top-$k$ structure, using randomness to get there dramatically faster.

## The algorithm (pseudocode)

Given $A$ ($m \times n$), a target rank $k$, an oversampling parameter $p$ (commonly $p \approx 10$), and an optional number of power iterations $q$ (useful when the singular values decay slowly):

```
function randomized_svd(A, k, p = 10, q = 2):
    l = k + p                          # slightly oversample the target rank

    # 1. Random projection
    Omega = random_gaussian_matrix(n, l)     # n x l, iid standard normal entries
    Y = A @ Omega                            # m x l

    # 2. (Optional) power iterations to sharpen the projection
    for i in range(q):
        Y = A @ (A.T @ Y)
        Y = orthonormalize(Y)                # re-orthonormalize for numerical stability

    # 3. Build an orthonormal basis for the projected range of A
    Q, _ = qr_decomposition(Y)               # Q is m x l, orthonormal columns

    # 4. Project A onto this much smaller basis
    B = Q.T @ A                              # l x n  (small!)

    # 5. Exact SVD on the small matrix B (cheap, since l << m, n)
    U_tilde, Sigma, V = svd(B)

    # 6. Lift the result back up
    U = Q @ U_tilde                          # m x l

    # 7. Keep only the top k components
    return U[:, :k], Sigma[:k], V[:, :k]
```

## Why this works

Step 1 is the heart of the trick. Multiplying $A$ by a random matrix $\Omega$ produces $Y = A\Omega$, whose columns are random linear combinations of $A$'s columns. Because of how high-dimensional random projections behave (a consequence of concentration of measure), a *random* sample of just $l = k+p$ directions is, with very high probability, enough to capture the dominant column space of $A$ — provided $A$'s singular values decay reasonably fast, which is the typical case for real data such as text, images, or ratings matrices. The oversampling ($+p$) and the optional power iterations exist to make this probabilistic guarantee more reliable, especially when the singular value spectrum decays slowly and a plain random sample might miss part of the important structure.

Once you have $Q$, an orthonormal basis that approximates the range of $A$, everything downstream becomes cheap: $B = Q^TA$ is only $l \times n$ instead of $m \times n$, so computing its exact SVD is fast, and lifting $U_{\tilde{}}$ back up via $Q$ recovers an approximate SVD of the *original* matrix $A$ — but at a fraction of the cost of running exact SVD on $A$ directly.

## Applications

Randomized SVD is used anywhere you need a fast rank-$k$ approximation of a large matrix:

- **Large-scale LSI / topic modeling** — huge term-document matrices where only the top few latent concepts matter.
- **PCA on big datasets** — dimensionality reduction when the full covariance matrix is too large to decompose exactly.
- **Recommender systems** — huge, mostly-sparse user–item rating matrices, where a low-rank approximation reveals latent user/item factors.
- **Image and signal compression** — approximating large image or signal matrices with a small number of dominant components.

In every one of these cases, the pattern is the same: the data matrix is enormous, but the *useful* information sits in a much lower-dimensional subspace, and randomized SVD finds that subspace without paying the cost of a full decomposition.
